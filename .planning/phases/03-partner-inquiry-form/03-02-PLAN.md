This task runs after the user provides Resend credentials from Task 1.

1. **Deploy the Partner Worker** from workers/partner/:
   ```bash
   cd workers/partner
   npx wrangler deploy
   ```
   Note the deployed URL from the output. It will be like: `https://vbi-partner-inquiry.{subdomain}.workers.dev`

2. **Set Worker secrets** (using values from Task 1 checkpoint):
   ```bash
   echo "<RESEND_API_KEY_FROM_USER>" | npx wrangler secret put RESEND_API_KEY
   echo "<TO_EMAIL_FROM_USER>" | npx wrangler secret put TO_EMAIL
   ```

3. **Update PARTNER_WORKER_URL in js/main.js** if the actual deployed URL differs from the placeholder:
   - Find the line: `var PARTNER_WORKER_URL = 'https://vbi-partner-inquiry.naren-ekwa.workers.dev';`
   - Replace with the actual URL from step 1's deploy output
   - Use the same subdomain that was confirmed during Phase 2 deployment

4. **Test the Partner Worker directly with curl** (bypasses CORS):
   ```bash
   curl -X POST https://vbi-partner-inquiry.{subdomain}.workers.dev \
     -H "Content-Type: application/json" \
     -H "Origin: https://vbi.narenlife.com" \
     -d '{"company_name":"Test Corp","contact_name":"Jane Doe","contact_email":"jane@testcorp.com","contact_phone":"(555) 123-4567","category":"software","interest_level":"premier","message":"Test partner inquiry from deployment verification"}'
   ```
   Expected: `{"ok":true}` with HTTP 200

5. **Test validation error handling**:
   ```bash
   curl -X POST https://vbi-partner-inquiry.{subdomain}.workers.dev \
     -H "Content-Type: application/json" \
     -H "Origin: https://vbi.narenlife.com" \
     -d '{"company_name":"","contact_name":"","contact_email":"","contact_phone":"","category":"","interest_level":"","message":""}'
   ```
   Expected: `{"ok":false,"error":"All fields are required"}` with HTTP 400

6. **Test CORS preflight**:
   ```bash
   curl -X OPTIONS https://vbi-partner-inquiry.{subdomain}.workers.dev \
     -H "Origin: https://vbi.narenlife.com" \
     -H "Access-Control-Request-Method: POST" \
     -H "Access-Control-Request-Headers: Content-Type" \
     -v 2>&1 | grep -i "access-control"
   ```
   Expected: Access-Control-Allow-Origin: https://vbi.narenlife.com

7. **Push updated files to GitHub** (partners/index.html + js/main.js changes from Plan 01, plus any URL fix):
   ```bash
   git add partners/index.html js/main.js
   git commit -m "Add partner inquiry form submission to Cloudflare Worker"
   git push
   ```
   Note: Workers directory is NOT pushed to the main site repo. Only the HTML/JS changes go to GitHub Pages.

8. **Verify email delivery**: Check the recipient inbox (TO_EMAIL) for the test email from step 4. Subject should be "Partner Inquiry: Test Corp - software".

9. **Verify separation from contact form**: The partner email should:
   - Have subject starting with "Partner Inquiry:" (not "Contact Form:")
   - Come from partners@narenlife.com (not contact@narenlife.com)
   - Arrive at the partner TO_EMAIL (not the contact form TO_EMAIL, if they differ)
