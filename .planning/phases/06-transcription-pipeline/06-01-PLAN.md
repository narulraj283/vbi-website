  <files>scripts/requirements.txt, scripts/transcribe.py</files>
  <action>
    1. Create `scripts/requirements.txt` with:
       - faster-whisper==1.1.0 (1.2.1 does not exist on PyPI -- use 1.1.0 which is the latest stable release with VAD support via silero-vad)
       - beautifulsoup4>=4.12
       - requests>=2.31

    2. Install dependencies: `pip3 install -r scripts/requirements.txt`

    IMPORTANT: faster-whisper 1.2.1 does not exist on PyPI. The latest stable version is 1.1.0. Use that. If pip install fails for 1.1.0 as well, fall back to the latest available version. Check `pip3 index versions faster-whisper` or just `pip3 install faster-whisper` to get whatever is latest.

    3. Create `scripts/transcribe.py` — a single Python script that does the full pipeline:

    **CLI interface:**
    ```
    python3 scripts/transcribe.py --episode N [--inject] [--model small] [--output-dir transcripts]
    ```
    - `--episode N` (required): Episode number (1-101)
    - `--inject`: If set, also inject the transcript HTML into the episode page
    - `--model`: Whisper model size (default: "small" — good balance of speed and accuracy for CPU)
    - `--output-dir`: Directory for transcript HTML fragments (default: "transcripts")

    **Step A — Extract audio URL:**
    Parse `podcast/episode-{N}/index.html` using BeautifulSoup. Extract the Libsyn MP3 URL from the `<source>` tag inside the `<audio>` element. Do NOT hardcode the URL pattern — read it from the HTML so the script works even if URLs vary.

    **Step B — Download audio:**
    Use `requests` to download the MP3 to a temp file. Stream the download (do not load entire file into memory). Show progress (file size). If the file already exists in a `cache/` directory, skip re-download.
    Create `scripts/cache/` directory for cached downloads.

    **Step C — Transcribe with faster-whisper + VAD:**
    ```python
    from faster_whisper import WhisperModel
    model = WhisperModel(model_size, device="cpu", compute_type="int8")
    segments, info = model.transcribe(
        audio_path,
        vad_filter=True,
        vad_parameters=dict(
            min_silence_duration_ms=500,
            speech_pad_ms=200,
        ),
        language="en",
        beam_size=5,
    )
    ```
    - Use `device="cpu"` and `compute_type="int8"` (this is a Mac without CUDA)
    - `vad_filter=True` activates silero-vad to skip silence/music (PODC-05)
    - Collect all segments with their start/end timestamps and text

    **Step D — Speaker labeling (simplified heuristic):**
    Since pyannote.audio requires a HuggingFace token and GPU for good performance, use a practical heuristic approach:
    - The VBI podcast is hosted by Naren Arulrajah, typically with one guest
    - Label the first speaker segment as "Host" (Naren)
    - Use a simple turn-detection heuristic: when there is a gap of >2 seconds between segments OR a significant change in segment length pattern, toggle between Host and Guest
    - Format: `<p><strong>Host:</strong> segment text</p>` or `<p><strong>Guest:</strong> segment text</p>`
    - For episodes that appear to be monologues (few speaker changes detected), just use "Host:" for all segments
    - Add a comment at the top of transcript output noting this uses heuristic labeling

    **Step E — Generate HTML fragment:**
    Build a transcript section as clean HTML:
    ```html
    <div class="transcript-section" id="transcript">
        <h2>Episode Transcript</h2>
        <div class="transcript-content">
            <p><strong>Host:</strong> escaped segment text...</p>
            <p><strong>Guest:</strong> escaped segment text...</p>
            ...
        </div>
    </div>
    ```
    - HTML-escape ALL transcript text using `html.escape()` from Python stdlib (this is simpler and more correct than BeautifulSoup for escaping — BeautifulSoup is used for HTML parsing/injection, not escaping)
    - Also use BeautifulSoup's `new_tag` and `new_string` methods when building the HTML to ensure proper escaping (PODC-06)
    - Group consecutive same-speaker segments into single paragraphs (avoid one `<p>` per 5-second chunk)
    - Write the fragment to `transcripts/episode-{N}.html`

    **Step F — Inject into episode page (when --inject is passed):**
    - Read `podcast/episode-{N}/index.html` with BeautifulSoup (html.parser)
    - Find the `<div class="episode-main">` element
    - Find the last child div (the "Perfect For" section) inside episode-main
    - Insert the transcript section div AFTER the "Perfect For" div
    - Also add minimal CSS for the transcript section inline in the existing `<style>` block:
      ```css
      .transcript-section { margin-top: 2rem; padding-top: 2rem; border-top: 2px solid #E8913A; }
      .transcript-section h2 { color: #1B2A4A; margin-bottom: 1rem; }
      .transcript-content p { margin-bottom: 0.75rem; line-height: 1.8; }
      .transcript-content strong { color: #0D7377; }
      ```
    - Write the modified HTML back to `podcast/episode-{N}/index.html`
    - Use `formatter=None` in BeautifulSoup's prettify/output to avoid reformatting the entire HTML

    **Error handling:**
    - If audio download fails (404, network error), print clear error and exit 1
    - If transcription produces zero segments, warn about possible audio issue
    - If episode HTML file not found, print error and exit 1

    **Print summary on completion:**
    - Episode number, audio URL, audio duration
    - Number of segments transcribed
    - Approximate word count
    - Number of speaker changes detected
    - Output file path(s)
