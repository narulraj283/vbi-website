  <name>Task 1: Create shared security module with Turnstile, honeypot, and rate limiting</name>
  <files>workers/shared/security.js</files>
  <action>
Create `workers/shared/security.js` exporting three functions:

1. `async function verifyTurnstile(token, secretKey, ip)` — POST to `https://challenges.cloudflare.com/turnstile/v0/siteverify` with `{ secret: secretKey, response: token, remoteip: ip }`. Return `{ ok: true }` if response.success is true, otherwise `{ ok: false, error: "Bot verification failed" }`. If token is falsy, return `{ ok: false, error: "Missing verification token" }` immediately (no fetch).

2. `function isHoneypot(body)` — Check if `body._gotcha` is truthy (non-empty string). Return true if honeypot is filled (bot detected). The field name `_gotcha` is the honeypot — real users never fill it because it is hidden via CSS.

3. `async function checkRateLimit(env, ip, limit = 5, windowSeconds = 3600)` — Use KV namespace bound as `env.RATE_LIMIT`. Key format: `ip:{ip}`. On each call: get current count from KV; if count >= limit, return `{ ok: false, remaining: 0 }`; otherwise put incremented count with expirationTtl of windowSeconds and return `{ ok: true, remaining: limit - count - 1 }`. If key does not exist, treat as count 0. Store count as a plain number string in KV.

Export all three as named exports.
  </action>

  </verify>

**For each of the three Workers (contact, partner, newsletter), apply the same pattern:**

In `src/index.js`:
1. Add import at top: `import { verifyTurnstile, isHoneypot, checkRateLimit } from "../../shared/security.js";`
2. After parsing the JSON body and BEFORE any field validation, add these three checks in order:

```javascript
// --- Security checks ---
// 1. Honeypot (silent reject — return fake success to confuse bots)
if (isHoneypot(body)) {
  return jsonResponse({ ok: true }, 200, env);
}

// 2. Turnstile verification
const turnstileResult = await verifyTurnstile(
  body["cf-turnstile-response"],
  env.TURNSTILE_SECRET_KEY,
  request.headers.get("CF-Connecting-IP")
);
if (!turnstileResult.ok) {
  return jsonResponse({ ok: false, error: turnstileResult.error }, 403, env);
}

// 3. Rate limiting
const rateResult = await checkRateLimit(env, request.headers.get("CF-Connecting-IP"));
if (!rateResult.ok) {
  return jsonResponse({ ok: false, error: "Too many submissions. Please try again later." }, 429, env);
}
```

3. For the newsletter Worker, add the security checks only in the `handleSubscribe` function (not in `handleConfirm` which is a GET endpoint).
