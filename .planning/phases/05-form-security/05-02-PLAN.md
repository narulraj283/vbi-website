  <action>
**For each of the three HTML pages, apply the same pattern:**

1. **Add Turnstile script** in the `<head>`:
```html
<script src="https://challenges.cloudflare.com/turnstile/v0/api.js" async defer></script>
```

2. **Add Turnstile widget** inside the `<form>`, just before the submit button:
```html
<div class="cf-turnstile" data-sitekey="SITE_KEY_PLACEHOLDER" data-callback="onTurnstileSuccess" data-theme="light"></div>
```
Use `SITE_KEY_PLACEHOLDER` as the data-sitekey value — the user will replace it with their actual site key from Cloudflare Dashboard after creating the Turnstile widget.

3. **Add honeypot field** inside the `<form>`, positioned where it will not be seen. Use CSS to hide it (not `display:none` or `type="hidden"` which smart bots detect):
```html
<div style="position:absolute;left:-9999px;" aria-hidden="true">
  <label for="_gotcha">Do not fill this</label>
  <input type="text" name="_gotcha" id="_gotcha" tabindex="-1" autocomplete="off">
</div>
```

4. **Update the form submission JavaScript** to include both fields in the POST body:
- Find the existing `fetch()` call that sends form data to the Worker
- Add `"cf-turnstile-response": document.querySelector('[name="cf-turnstile-response"]').value` to the JSON body
- Add `"_gotcha": document.getElementById("_gotcha").value` to the JSON body
- Note: The Turnstile widget automatically creates a hidden input named `cf-turnstile-response` when the user passes the challenge

5. **Add Turnstile callback** (optional UX improvement): Define `onTurnstileSuccess` function that enables the submit button. Start with submit button disabled (`disabled` attribute), enable it when Turnstile completes:
```javascript
function onTurnstileSuccess(token) {
  document.querySelector('button[type="submit"]').disabled = false;
}
```

**Important notes:**
- Do NOT remove any existing form fields or validation
- The Turnstile widget renders as a small checkbox-like element — place it where it fits the form layout
- For the newsletter form (likely in index.html footer), ensure the Turnstile widget fits the compact layout
  </action>
