  <action>
    Create `scripts/batch-transcribe.py` — a Python script that orchestrates running `scripts/transcribe.py` across all 101 episodes with robust error handling.

    **CLI interface:**
    ```
    python3 scripts/batch-transcribe.py [--start N] [--end N] [--model tiny|small] [--commit-every N] [--dry-run]
    ```
    - `--start N`: First episode to process (default: 1)
    - `--end N`: Last episode to process (default: 101)
    - `--model`: Whisper model size to pass through (default: "tiny" — use tiny for batch speed; small is 3-4x slower and batch would take many hours on CPU)
    - `--commit-every N`: Git commit after every N successful episodes (default: 10)
    - `--dry-run`: Print what would be done without executing

    **Core logic:**

    1. **Pre-flight checks:**
       - Verify `scripts/transcribe.py` exists
       - Verify `podcast/episode-{start}/index.html` through `podcast/episode-{end}/index.html` exist
       - Check that Python dependencies are installed (faster-whisper, beautifulsoup4, requests)
       - Print total episodes to process, estimated time warning

    2. **Skip detection (idempotency):**
       For each episode, before calling transcribe.py, check if `podcast/episode-{N}/index.html` already contains `class="transcript-section"`. If it does, log "Episode {N}: already has transcript, skipping" and move on. This makes the script safe to re-run.

    3. **Sequential processing loop:**
       For each episode N from start to end:
       - Print progress: `[{current}/{total}] Processing episode {N}...`
       - Run: `python3 scripts/transcribe.py --episode {N} --inject --model {model}`
       - Use `subprocess.run()` with `capture_output=True, text=True, timeout=1800` (30 min timeout per episode)
       - On success: log to results list, print brief summary
       - On failure (non-zero exit, timeout, exception): log error, print warning, continue to next episode
       - Track: successes, failures, skipped counts

    4. **Periodic git commits:**
       After every `--commit-every` successful transcriptions:
       - Run: `git add podcast/episode-*/index.html transcripts/`
       - Run: `git commit -m "feat(07): add transcripts for episodes {first}-{last} [{count}/{total}]"`
       - This ensures partial progress is saved even if the batch is interrupted
       - Use `subprocess.run()` for git commands too

    5. **Completion report:**
       At the end, print a summary table:
       ```
       === BATCH TRANSCRIPTION COMPLETE ===
       Total:    101
       Success:  {N}
       Skipped:  {N} (already had transcripts)
       Failed:   {N}

       Failed episodes: [list of episode numbers]
       ```
       Also write this report to `scripts/batch-log.txt` for reference.

    6. **Final git commit:**
       After all episodes processed, do one final commit for any remaining uncommitted work:
       - `git add podcast/episode-*/index.html transcripts/ scripts/batch-log.txt`
       - `git commit -m "feat(07): complete batch transcription — {success}/{total} episodes"`

    **Important implementation notes:**
    - Import subprocess, sys, os, argparse, datetime, pathlib
    - Use absolute paths based on script location: `PROJECT_ROOT = Path(__file__).resolve().parent.parent`
    - For the BeautifulSoup check (skip detection), import bs4 directly in this script
    - Print timestamps with each log line for timing analysis
    - Flush stdout after each print (`print(..., flush=True)`) so progress is visible in real-time
  </action>
