Since the only `"@type": "Podcast"` in episode files is inside partOfSeries (the episode itself uses "PodcastEpisode"), a global replace of `"@type": "Podcast"` (exact match, not "PodcastEpisode") is safe.

IMPORTANT: Use `sed` with the pattern `"@type": "Podcast"` and NOT `"@type": "Podcast` (without closing quote) -- the closing quote ensures we don't accidentally match "PodcastEpisode".

Exact sed for macOS:
```bash
for i in $(seq 1 101); do
  sed -i '' 's/"@type": "Podcast"/"@type": "PodcastSeries"/g' "podcast/episode-$i/index.html"
```

**Change 2: Convert datePublished from human-readable to ISO 8601**

Current format: `"datePublished": "October 06, 2023",`
Target format: `"datePublished": "2023-10-06",`

Write a script that for each episode file:
1. Extracts the datePublished value (e.g., "October 06, 2023")
2. Converts month name to number: January=01, February=02, ..., December=12
3. Rearranges to YYYY-MM-DD format
4. Replaces in the file using sed

Example approach (bash):
```bash
for i in $(seq 1 101); do
  FILE="podcast/episode-$i/index.html"
  # Extract the date string
  DATE_STR=$(grep '"datePublished"' "$FILE" | sed 's/.*"datePublished": "//;s/".*//')
  # Parse month, day, year
  MONTH_NAME=$(echo "$DATE_STR" | awk '{print $1}')
  DAY=$(echo "$DATE_STR" | awk '{print $2}' | tr -d ',')
  YEAR=$(echo "$DATE_STR" | awk '{print $3}')
  # Convert month name to number
  case $MONTH_NAME in
    January) MM=01;; February) MM=02;; March) MM=03;; April) MM=04;;
    May) MM=05;; June) MM=06;; July) MM=07;; August) MM=08;;
    September) MM=09;; October) MM=10;; November) MM=11;; December) MM=12;;
  esac
  # Pad day
  DAY=$(printf "%02d" "$DAY")
  ISO_DATE="$YEAR-$MM-$DAY"
  # Replace in file
  sed -i '' "s/\"datePublished\": \"$DATE_STR\"/\"datePublished\": \"$ISO_DATE\"/" "$FILE"
done
```
