  <action>
This task runs after the user provides Resend credentials from Task 1.

1. **Deploy the Worker** from workers/newsletter/:
   ```bash
   cd workers/newsletter
   npx wrangler deploy
   ```
   Note the deployed URL from the output (e.g., `https://vbi-newsletter.naren-ekwa.workers.dev`).

2. **Update CONFIRM_URL in wrangler.toml** to match the actual deployed URL:
   - Set `CONFIRM_URL = "https://vbi-newsletter.naren-ekwa.workers.dev"` (or whatever the actual URL is)
   - Redeploy: `npx wrangler deploy` (so the Worker knows its own URL for building confirmation links)

3. **Set Worker secrets** (using values from Task 1 checkpoint):
   ```bash
   echo "<RESEND_API_KEY_FROM_USER>" | npx wrangler secret put RESEND_API_KEY
   echo "<RESEND_AUDIENCE_ID_FROM_USER>" | npx wrangler secret put RESEND_AUDIENCE_ID
   echo "<CONFIRM_SECRET_FROM_USER>" | npx wrangler secret put CONFIRM_SECRET
   ```

4. **Update WORKER_URL in js/main.js** if the actual deployed URL differs from the placeholder:
   - Find: `var WORKER_URL = 'https://vbi-newsletter.naren-ekwa.workers.dev';`
   - Replace with the actual URL from step 1's deploy output if different

5. **Test POST /subscribe with curl**:
   ```bash
   curl -X POST https://vbi-newsletter.naren-ekwa.workers.dev/subscribe \
     -H "Content-Type: application/json" \
     -H "Origin: https://vbi.narenlife.com" \
     -d '{"email":"<USER_TEST_EMAIL>"}'
   ```
   Use a REAL email address the user can check. Expected response: `{"ok":true,"message":"Check your email to confirm your subscription"}`

6. **Test validation error**:
   ```bash
   curl -X POST https://vbi-newsletter.naren-ekwa.workers.dev/subscribe \
     -H "Content-Type: application/json" \
     -H "Origin: https://vbi.narenlife.com" \
     -d '{"email":""}'
   ```
   Expected: `{"ok":false,"error":"Email is required"}` with HTTP 400

7. **Test CORS preflight**:
   ```bash
   curl -X OPTIONS https://vbi-newsletter.naren-ekwa.workers.dev/subscribe \
     -H "Origin: https://vbi.narenlife.com" \
     -H "Access-Control-Request-Method: POST" \
     -H "Access-Control-Request-Headers: Content-Type" \
     -v 2>&1 | grep -i "access-control"
   ```
   Expected: Access-Control-Allow-Origin: https://vbi.narenlife.com

8. **Verify confirmation email arrived**: Check the test email inbox for "Confirm your VBI newsletter subscription" from `VBI Newsletter <newsletter@narenlife.com>`. The email should contain a clickable confirmation link.

9. **Test the confirmation link**: Click the confirmation link in the email (or curl it). It should return an HTML page saying "You're subscribed!" and the subscriber should appear in the Resend audience.

10. **Push updated files to GitHub**:
    ```bash
    git add index.html js/main.js
    git commit -m "Add newsletter signup form with Worker submission"
    git push
    ```
    Note: workers/ directory is NOT pushed to the site repo. Only HTML/JS changes go to GitHub Pages.
