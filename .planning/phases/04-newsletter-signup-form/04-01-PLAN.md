**workers/newsletter/src/index.js:**
Reuse the corsHeaders(env) and jsonResponse(body, status, env) helper pattern from the contact worker exactly. Then implement routing:

The Worker handles two routes:

1. **POST /subscribe** (called by frontend form):
   - Parse JSON body, extract `email`
   - Validate email is present and matches `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`
   - Generate a signed confirmation token: create a JSON payload `{email, exp: Date.now() + 86400000}` (24h expiry), then use Web Crypto `crypto.subtle.sign("HMAC", key, data)` with the CONFIRM_SECRET env var as the HMAC key. Encode the payload as base64url and the signature as base64url, join with `.` (format: `base64url(payload).base64url(signature)`)
   - Build confirmation URL: `${env.CONFIRM_URL}/confirm?token=${token}` where CONFIRM_URL is a [vars] entry set to the Worker's own URL (e.g., `https://vbi-newsletter.naren-ekwa.workers.dev`)
   - Send confirmation email via Resend:
     - from: `VBI Newsletter <${env.FROM_EMAIL}>`
     - to: the submitted email
     - subject: "Confirm your VBI newsletter subscription"
     - html: Simple HTML email with: "Thanks for subscribing to the VBI newsletter! Please confirm your email by clicking the link below:" followed by the confirmation URL as a styled button/link. Include "This link expires in 24 hours." and "If you didn't request this, you can ignore this email."
   - Return `{ok: true, message: "Check your email to confirm your subscription"}` with 200
   - On Resend error: return `{ok: false, error: "Failed to send confirmation email. Please try again."}` with 500

2. **GET /confirm?token=...** (called when user clicks email link):
   - Extract `token` from URL search params
   - Split token on `.` to get payload and signature parts
   - Decode base64url payload, parse JSON to get `{email, exp}`
   - Verify HMAC signature using `crypto.subtle.verify("HMAC", key, signature, data)` with CONFIRM_SECRET
   - Check expiry: if `exp < Date.now()`, return an HTML page saying "This confirmation link has expired. Please sign up again."
   - If valid: add subscriber to Resend audience using `resend.contacts.create({email, audienceId: env.RESEND_AUDIENCE_ID})`
   - Return an HTML page (not JSON — this is a browser click): "You're subscribed! Welcome to the VBI newsletter." with a link back to the homepage.
   - On error: return HTML page "Something went wrong. Please try signing up again." with a link to homepage.

**HMAC key import helper:** Create an async function `getHmacKey(secret)` that uses `crypto.subtle.importKey("raw", new TextEncoder().encode(secret), {name: "HMAC", hash: "SHA-256"}, false, ["sign", "verify"])`.

**Base64url helpers:** Create `toBase64Url(buffer)` and `fromBase64Url(str)` functions using btoa/atob with URL-safe character replacements (+→-, /→_, remove =).

**CORS note:** POST /subscribe needs CORS headers (browser fetch). GET /confirm does NOT need CORS (browser navigation, returns HTML). Apply corsHeaders(env) only to POST and OPTIONS responses.

```javascript
function initNewsletterForm() {
  var newsletterForm = document.querySelector('.newsletter-form');
  if (!newsletterForm) return;

  var WORKER_URL = 'https://vbi-newsletter.naren-ekwa.workers.dev';
  var feedback = document.getElementById('newsletter-feedback');

  newsletterForm.addEventListener('submit', async function(e) {
    e.preventDefault();

    var emailInput = newsletterForm.querySelector('input[name="email"]');
    if (!emailInput) return;

    var email = emailInput.value.trim();

    if (!email) {
      showNewsletterFeedback(feedback, 'Please enter your email address.', 'error');
      return;
    }

    if (!isValidEmail(email)) {
      showNewsletterFeedback(feedback, 'Please enter a valid email address.', 'error');
      return;
    }

    var submitBtn = newsletterForm.querySelector('button[type="submit"]');
    var originalText = submitBtn.textContent;
    submitBtn.disabled = true;
    submitBtn.textContent = 'Subscribing...';
    feedback.textContent = '';
    feedback.className = '';

    try {
      var res = await fetch(WORKER_URL + '/subscribe', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ email: email }),
      });
      var result = await res.json();

      if (result.ok) {
        showNewsletterFeedback(feedback, 'Check your email to confirm your subscription!', 'success');
        emailInput.value = '';
      } else {
        showNewsletterFeedback(feedback, result.error || 'Something went wrong. Please try again.', 'error');
      }
    } catch (err) {
      showNewsletterFeedback(feedback, 'Network error. Please check your connection and try again.', 'error');
    } finally {
      submitBtn.disabled = false;
      submitBtn.textContent = originalText;
    }
  });
}

function showNewsletterFeedback(el, message, type) {
  el.textContent = message;
  el.className = type === 'success' ? 'newsletter-success' : 'newsletter-error';
}
```

This follows the exact same pattern as initContactForm() — var declarations (not const/let for consistency), async submit handler, loading state on button, fetch POST with JSON, feedback display, error handling.

The old `showNewsletterError()` function is removed and replaced by `showNewsletterFeedback()` which handles both success and error states.
