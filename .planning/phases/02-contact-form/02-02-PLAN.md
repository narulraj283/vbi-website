<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Set up Resend account and verify narenlife.com domain</name>
  <what-built>The Worker code and client JS are complete from Plan 01. Before deploying, Resend must be configured so the Worker can actually send email.</what-built>
  <how-to-verify>
    Complete these steps:

    1. **Create Resend account** (skip if you already have one):
       - Go to https://resend.com/signup
       - Sign up with any email

    2. **Add narenlife.com domain in Resend**:
       - Go to Resend Dashboard -> Domains -> Add Domain
       - Enter: `narenlife.com`
       - Resend will show DNS records to add (SPF TXT, DKIM CNAME, optional DMARC TXT)

    3. **Add DNS records in Cloudflare** (account: Naren@ekwa.com):
       - Go to Cloudflare Dashboard -> narenlife.com -> DNS -> Records
       - Add the SPF TXT record Resend provides
       - Add the DKIM CNAME record(s) Resend provides
       - Add the DMARC TXT record if Resend provides one
       - Wait for Resend to show domain status as "Verified" (usually 1-5 minutes with Cloudflare)

    4. **Create Resend API key**:
       - Go to Resend Dashboard -> API Keys -> Create API Key
       - Name: "vbi-contact-form"
       - Permission: Full access
       - Copy the key (starts with `re_`)

    5. **Decide on recipient email** (TO_EMAIL):
       - What email address should receive contact form submissions?

    When done, share:
    - Resend API key (re_...)
    - Recipient email address (TO_EMAIL)
    - Confirmation that narenlife.com domain is verified in Resend
  </how-to-verify>
  <resume-signal>Provide: Resend API key, recipient email, and confirm domain is verified. Or type "skip" to deploy without secrets (Worker will fail to send email but will be reachable).</resume-signal>
</task>

  <action>
This task runs after the user provides Resend credentials from Task 1.

1. **Deploy the Worker** from workers/contact/:
   ```bash
   cd workers/contact
   npx wrangler deploy
   ```
   Note the deployed URL from the output. It will be like: `https://vbi-contact-form.naren-ekwa.workers.dev`

2. **Set Worker secrets** (using values from Task 1 checkpoint):
   ```bash
   echo "<RESEND_API_KEY_FROM_USER>" | npx wrangler secret put RESEND_API_KEY
   echo "<TO_EMAIL_FROM_USER>" | npx wrangler secret put TO_EMAIL
   ```

3. **Update WORKER_URL in js/main.js** if the actual deployed URL differs from the placeholder:
   - Find the line: `var WORKER_URL = 'https://vbi-contact-form.naren-ekwa.workers.dev';`
   - Replace with the actual URL from step 1's deploy output
   - The subdomain is determined by the Cloudflare account

4. **Test the Worker directly with curl** (bypasses CORS):
   ```bash
   curl -X POST https://vbi-contact-form.naren-ekwa.workers.dev \
     -H "Content-Type: application/json" \
     -H "Origin: https://vbi.narenlife.com" \
     -d '{"name":"Test User","email":"test@example.com","subject":"general","message":"Test message from deployment verification"}'
   ```
   Expected: `{"ok":true}` with HTTP 200

5. **Test validation error handling**:
   ```bash
   curl -X POST https://vbi-contact-form.naren-ekwa.workers.dev \
     -H "Content-Type: application/json" \
     -H "Origin: https://vbi.narenlife.com" \
     -d '{"name":"","email":"","subject":"","message":""}'
   ```
   Expected: `{"ok":false,"error":"All fields are required"}` with HTTP 400

6. **Test CORS preflight**:
   ```bash
   curl -X OPTIONS https://vbi-contact-form.naren-ekwa.workers.dev \
     -H "Origin: https://vbi.narenlife.com" \
     -H "Access-Control-Request-Method: POST" \
     -H "Access-Control-Request-Headers: Content-Type" \
     -v 2>&1 | grep -i "access-control"
   ```
   Expected: Access-Control-Allow-Origin: https://vbi.narenlife.com

7. **Push updated files to GitHub** (contact/index.html + js/main.js changes from Plan 01, plus any URL fix):
   ```bash
   git add contact/index.html js/main.js
   git commit -m "Add contact form submission to Cloudflare Worker"
   git push
   ```
   Note: Workers directory is NOT pushed to the main site repo (it's a separate Cloudflare deployment). Only the HTML/JS changes go to GitHub Pages.

8. **Verify email delivery**: Check the recipient inbox (TO_EMAIL) for the test email from step 4. Subject should be "Contact Form: general - Test User".
